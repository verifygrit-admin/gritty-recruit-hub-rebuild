Sprint 027 Session Spec

I want to run a single development sprint in sprint mode. This will be Sprint 027 in GrittyFB Recruit Hub, dedicated to creating new Admin Panel tab called "Account Updates" with eight subsequent data category toggles for student profiles, high school coach accounts, counselor accounts, high school accounts, college profiles, college coach profiles, and recruiting events, each with their own "Account Update" form.

There will be five goals for this sprint that, when achieved, will constitute a Done State: 

1. The "Account Updates" tab will allow the admin account, chris@grittyfb.com, to have read-write access to any  student profiles, high school coach accounts, counselor accounts, high school accounts, college profiles, college coach profiles, and recruiting event table data field, except for the user id, uuid, account email (if present), or account password (if present), for which the Admin will only have read access.

2. The "Account Updates" tab will only be available in Admin View.

3. The five "Account Update" tab forms will be modeled on the functionality of the "Player Updates" form in Coach View, with the following exceptions:

(A) The "Account Updates" form will only allow for ten accounts within one category of the data table to be edited at any one time.
(B) The "Account Updates" form functionality will NOT require an additional layer of verification and approval since the data updates are already being created and verified by the Admin through the "Account Updates" form.
(C) The "Account Updates" tab will contain toggles between eight different types of account "Account Update" forms (student profiles, high school coach accounts, counselor accounts, high school accounts, college profiles, college coach profiles, and recruiting events). This will essentially result in eight "Account Update" forms separate by toggles, each for one of the eight account types.

4. The "Account Updates" form should also allow for the creation or deletion of college profiles, college coach profiles, and recruiting events since these are not associated with an auth layer. However, no such deletion/creation option should be made available for student profiles, high school coach accounts, counselor accounts, or high school accounts, since these are auth-associated.

5. The Admin should be able to:

(A) Log into Admin View
(B) Navigate to the "Account Updates" panel/tab
(C) Select an account type toggle for either "Students" | "HS Coaches" | "Counselors" | "High Schools" | "Colleges" | "College Coaches" | "Recruiting Events" Account Update Forms
(D) Select up to 10 associated accounts from that category of Update Form
(E) Add missing data or edit existing data to those accounts selected for editing
(F) Select Update > Review Data Being Update > Confirm and Submit the Updated Account Data
